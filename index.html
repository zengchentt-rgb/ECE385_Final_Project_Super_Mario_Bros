üîß CHALLENGES SOLVED üîß

üíæ LUT Resource Constraints

Initial fire flower implementation exceeded FPGA LUT budget (~35,000 LUTs vs 32,600 available), causing synthesis failures.

Removed fire flower feature, optimized enemy arrays from 10 to 3 per type, reduced level data storage, converted sprite functions to BRAM ROMs.

üé® Sprite ROM Organization

Initial combinational logic approach for sprites caused timing failures and excessive LUT usage.

Migrated to BRAM-based ROMs with procedural initialization, implemented 2-stage rendering pipeline to handle BRAM read latency.

‚è±Ô∏è Clock Domain Crossing

Direct connection between SD card (95 MHz burst) and audio output (22 kHz continuous) caused data corruption and audio glitches.

Implemented Xilinx FIFO Generator IP with independent clocks, added Gray code pointers for safe cross-domain synchronization.

üíø SD Card Initialization

SD cards sometimes failed CMD0 command, leaving system in undefined state with no audio playback.

XESS controller sends 160 clock pulses before CMD0, retries ACMD41 until card leaves IDLE state, added timeout handling.

üîÄ Mixed VHDL/SystemVerilog

Signal type mismatches between VHDL (std_logic) and SystemVerilog (logic) caused compilation errors.

Created wrapper modules with explicit port matching, used common.vhd package for type definitions, careful signal width matching.

‚å®Ô∏è Keyboard Input Timing

Keys registering multiple times per press, causing uncontrollable jumps and menu navigation issues.

Implemented edge detection with prev_keycode register, only respond on rising edge of key detection, added debouncing.

üí• Collision Detection Bugs

Enemies passing through pipes, players getting stuck in platforms, inconsistent stomp detection.

Refined AABB collision bounds, added separate checks for horizontal/vertical collisions, implemented collision priority system.
